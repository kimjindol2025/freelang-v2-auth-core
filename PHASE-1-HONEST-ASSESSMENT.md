### Task 1.2: Block Parser (21 tests)

| 항목 | 평가 | 근거 |
|------|------|------|
| **테스트 통과** | 21/21 ✅ | 현재 모두 passing |
| **실제 기능** | ❌ 버그 발견 | off-by-one 오류 |
| **안정성** | 🔴 낮음 | 라인 계산 오류 |
| **품질** | 50/100 | 버그 있음 |

**발견된 버그**:

1. **`isInBlock(line)` - Line 347**:
   ```typescript
   b.line < line && b.line + b.body.length >= line
   ```
   - **문제**: `b.body.length` (Statement 배열 크기) ≠ 실제 라인 수
   - **영향**: 네스팅된 블록에서 라인 판정 오류
   - **예**:
     ```
     Line 0: if x > 5
     Line 1:   y = 10
     Line 2:   z = 20
     ```
     - Statement 배열: [stmt0, stmt1, stmt2] → length = 3
     - `b.line + b.body.length = 0 + 3 = 3` → 라인 2는?
     - ❌ off-by-one 오류

2. **`getBlockAt(line)` - Line 339**:
   ```typescript
   return this.blocks.find(b => b.line === line);
   ```
   - 정확도: Statement 라인과 소스 코드 라인이 다르면 실패
   - 네스팅 블록에서 의도하지 않은 매칭 가능

**원래 테스트가 감지 못한 이유**:
```typescript
// tests/phase-1-task2.test.ts Line 282
expect(analyzer.startsBlock(1)).toBe(true);  // ← 너무 간단한 검사
// 실제로는 off-by-one이 여기서 드러남
```

**결론**: 기본 구조는 있지만 라인 계산에 심각한 버그

| **품질** | 65/100 | 기본 동작만 |

**구현 문제**:

1. **Regex 기반 단순 매칭** (type-inference.ts Line 173-175):
   ```typescript
   const comparisonOps = ['>', '<', '==', '!=', '>=', '<='];
   if (comparisonOps.some(op => intent.includes(op))) {
     return 'bool';
   }
   ```
   - ❌ "x > y + 5" → "bool" (정확)
   - ⚠️ "print(x > 5)" → context 무시
   - ❌ "result = x > 5 ? a : b" → 실패 가능

2. **`inferExpressionType` - 부정확**:
   - "hello" + "world" → 문자열 (정확)
   - "5" + "10" → 어느 것? (미정의)
   - "x + y" where x=string, y=number → 추론 불가

3. **Type overwrite 버그**:
   - 여러 번 호출하면 이전 타입을 덮어쓸 수 있음
   - 상태 관리 미흡

**테스트가 감지 못한 이유**:
```typescript
// 테스트가 "타입 존재" 여부만 확인
expect(stub.returnType).toBe('number');  // ← 단순 체크
// 정확도 검증 없음
```

**결론**: 기본 동작만 하고, 복잡한 타입 추론은 실패 가능
